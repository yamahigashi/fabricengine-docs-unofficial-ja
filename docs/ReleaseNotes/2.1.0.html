Fabric Engine 2.1.0 リリースノート¶

Fabric Engine version 2.1.0
2016年2月吉日リリース
Copyright (c) 2010-2017 Fabric Software Inc. All rights reserved.

このリリースについて¶

Fabric Engine 2.1.0 is the first feature release since the introduction of Canvas in Fabric Engine 2. We’ve been carefully listening feedback and have focused on making your Fabric Engine experience – whether you are just learning or are building complex tools – more intuitive and productive. Here are some highlights:

  • The Value Editor has been completely rewritten and in doing so we’ve added some pretty convenient things like spin boxes and the ability to skin it using QSS.

  • 『初めてFabricに触るプログラマが知るべきこと』についての専用のドキュメントセクションを追加しました。 http://fabricengine.com/fabric-101-for-programmers/ をご覧ください。

  • presets, サンプルを豊富に追加しました。詳細は後述します。

  • Fabric for Maya へ『CanvasDeformer』ノードを追加しました。通常の『CanvasNode』より 4-5倍高速です。

  • 診断ウィンド（diagnostic window)を追加しました。グラフ上でのエラーや接続ミスを、より容易に素早くトラブルシュート可能です。

  • Coreでの多くの問題を解決しました。

  • DFGBindings: KLから Canvasバインディングを実行することができるようになりました。これを <graphs flowing through the data graph> を略し DFGBinding と呼称します。 http://fabricengine.com/graphs-flowing-through-graphs/ により詳しい説明があります。

  • 200以上のバグを修正、機能向上や使い勝手の向上をおこないました。例えば .canvas 拡張子を持つファイルのドラッグ・アンド・ドロップの許容などを含みます。

サポートするOS, DCC統合, GPU¶

  • Fabric 2 は Windows, Linux をサポートします。OS X はバイナリビルドが提供されますが公式にはサポートされません。ただしこれからのリリースでサポート予定です。

  • Fabric for Maya 統合では Windows/Linux 版 Autodesk Maya 2014, 2015, 2016 （とそれぞれのサービスパック）をサポートします。OS X はバイナリビルドが提供されますが公式にはサポートされません。ただしこれからのリリースでサポート予定です。サンプルシーンを Maya 2016 上で使用する際、Parallel Evaluation を有効にしているといくつかの不具合が起きます。この問題は Maya 2016 SP4 で解決されています。

  • Fabric for Softimage 統合版は Autodesk Softimage 2014、2015（SP版含むWindows版）をサポートします。

  • Fabric 2 GPU supportは NVIDIAハードウェアのみ利用可能です。詳細はGPUコンピューティングガイドを御覧ください。

サポート¶

Fabric 2.1 は評価版、スタジオ版、教育版が無償で利用可能です。また商用契約に基づく利用も可能です。商用ライセンスをご利用のお客様は、私達Fabricへ直接連絡いただけます。非商用のお客様はサポートについて以下のオプションをとることをお勧めいたします。

  • http://forums.fabricengine.com :Fabric コミュニティによる Ask & answer questions。サポートに関する問題を投稿する場です。

  • mailto://bugs@fabricengine.com : もしバグを発見したら、明快な再現手段を付しEメールをください。(使用OSとDCCアプリ、それぞれのバージョンについての詳細も)Fabricからの返答は、Fabricプラットフォームの問題の解決にあたって追加情報を必要とする場合のみですので、ご了承ください。

変更点¶

注釈

EDK の関数 isValid() は klObjectIsValid(); に変更しました。isValid()使用箇所を必要に応じ修正し、再コンパイルしてください。

注釈

Compounds/PolygonMesh/Queries 以下のプリセット, Conversion プリセットや、Query プリセットは、それぞれのサブグラフにおいては可読性のため、私達のガイドラインに従い新しく追加された Attribute プリセットを使用するようにしてください。これは将来のリリースでかいけつ予定です。

DFGBindings

Canvas bindings を KLから実行することが可能になりました。Canvas グラフを『オブジェクト』としてグラフ内で引き回すことができます。これにより、再利用性が飛躍的にまし、大規模なシステムへとスケールする柔軟性を手にしました。DFGBinding は 『グラフ』とその『データ』を持ちます。『データ』とはすなわち『入力』であり、有用な作業を行うものに必要なものであり、また作業の『結果』でもあります。したがってFabricに関するかぎり DFGBinding はデータの一部であり、他のデータと同様グラフ中で流れさせる事が可能です。これは以下を意味します。グラフ内にグラフ <ディスクから単体で読み込んできものや、さらには配列や辞書に格納したものまで> を作成できます。http://fabricengine.com/graphs-flowing-through-graphs/ により詳しい説明とビデオがあります。

canvasDeformer ノード

Fabric for Maya に新しいノード、canvasDeformer を追加しました。Maya のデフォメーションツールとコマンドを活用し、トポロジを変えること無くPolygonMesh の位置配列を編集します。新規のメッシュの作成を避けることができるので CanvasNode によるデフォメーションは4-5倍高速になります。
