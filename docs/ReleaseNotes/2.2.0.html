Fabric Engine 2.2.0 リリースノート¶

Fabric Engine version 2.2.0
2016年5月吉日リリース
Copyright (c) 2010-2017 Fabric Software Inc. All rights reserved.

このリリースについて¶

Fabric Engine 2.2.0 では多数の微妙な挙動とバグの修正と同時に、アクセシビリティ向上とよく聞かれる機能への要望の実現を目指しました。どれも Fabric体験をよりよくするものです。以下にハイライトを記します。 What’s New In Fabric Engine 2.2 video クイックチュートリアルもご覧ください。

  • Python! Canvas をわかりやすい専用アプリとして Python版として作り直しました。この新しい Python版Canvas のソースコードを用いることで、Canvas グラフの作成と編集を C++抜きに自動化することができます。

  • ごく簡単な Alembicアセットのビューアアプリケーションをの参考として canvas.py へ変更を加えたり新しいアプリの作成をどのように行うかのサンプルを同梱しています。以下に詳細をご覧ください。

  • Script Editor を統合、Pythonのシンタックスハイライトが搭載され、スタンドアロン版 canvas.py でのスクリプトの編集に大変便利です。

  • KLプログラマの皆さんのため、Canvas内でのコードコンプリーションも加えました。これからさらにどのように改善をするかのアイデアがまだまだたくさんあります。有用であると判断したらすぐにでも組み込みます。

  • Fabric Engine 2.2 の起動が2倍速くなりました。

  • ドラッグアンドドロップによるポートの順序入れ替えを、要望にお応えして追加しました。気に入っていただけますように。

  • Execute Ports により Canvasグラフにおけるポートの実行を制御できます。例示すると、 “If” ノードの “then” “else” に execute port を接続し、グラフにおける2つの部品どちらを実行するか分岐、決定することができます。単純なループもサポートしました。Canvas でのより完全なループ制御ソリューションは鋭意作成中です。

  • Fabric for MODO! ここに MODO統合版が Betaから公式にサポートへと昇格し、Fabric Engine 2.2のディストリビューションに含まれることを喜んでお伝えします。Fabric for MODO プラグインは MODO 901 から MODO 10.0と互換性があり、Windows, Linux, OS Xで利用可能です。もちろん Fabricでビルドしたあなたのツールは Mayaでも Softimageでも MODOでも動きます。

  • 250以上の障害を修正しました（特記事項は以下に掲載）

サポートするOS, DCC統合, GPU¶

  • Fabric 2.2 は Windows, Linux をサポートします。OS X はバイナリビルドが提供されますが公式にはサポートされません。ただしこれからのリリースでサポート予定です。

  • Fabric for Maya 統合では Windows/Linux 版 Autodesk Maya 2014, 2015, 2016 （とそれぞれのサービスパック）をサポートします。Maya 2016 Extenios 2 はサポートのための期間が取れなかったため含まれていません。デイリービルドにて対応予定です。OS X はバイナリビルドが提供されますが公式にはサポートされません。ただしこれからのリリースでサポート予定です。サンプルシーンを Maya 2016 上で使用する際、Parallel Evaluation を有効にしているといくつかの不具合が起きます。この問題は Maya 2016 SP4 で解決されています。

  • Fabric for Softimage 統合版は Autodesk Softimage 2014、2015（SP版含むWindows版）をサポートします。

  • Fabric for MODO 統合版は MODO 901 から MODO 10.0 まで Windows, Linux, OS X版を利用可能です。

  • Fabric 2 GPU supportは NVIDIAハードウェアのみ利用可能です。詳細はGPUコンピューティングガイドを御覧ください。

Fabric Engine 2 を取得するには http://dist.fabric-engine.com/FabricEngine/2.2.0/ から行えます。インストールについては Fabric Engine 入門 を参照してください。

サポート¶

Fabric 2.2 は評価版、スタジオ版、教育版が無償で利用可能です。また商用契約に基づく利用も可能です。商用ライセンスをご利用の顧客の皆様は、私達Fabricへ直接連絡いただけます。非商用のお客様はサポートについて以下のオプションをとることをお勧めいたします。

  • http://forums.fabricengine.com :Fabric コミュニティによる Ask & answer questions。サポートに関する問題を投稿する場です。

  • mailto://bugs@fabricengine.com : もしバグを発見したら、明快な再現手段を付しEメールをください。(使用OSとDCCアプリ、それぞれのバージョンについての詳細も)Fabricからの返答は、Fabricプラットフォームの問題の解決にあたって追加情報を必要とする場合のみですので、ご了承ください。

変更点¶

Python

新規機能の canvas.py のセットアップと動作についてのより詳細な情報は Fabricスタンドアロン版の起動(canvas.py) をご覧ください。

ごく簡単な Alembicアセットビューアのアプリケーションを用例として付属しています。 canvas.py のソースファイルをどのように編集し独自のアプリケーションを最小限の手間で作成するかの参考になるでしょう。 Fabric Apps With Python video のビデオがこの手順の演習となります。

Script Editor

新しい Script Editor にはシンタックスハイライトが搭載され Python コードを実行することができます。 Script Editor には Canvasでのグラフ操作を python コードとして反映しログに書き出すこともできます。 canvas.py に -s フラグを渡し実行することで有効になります。

KL Code Completion

KL エディタのコードウィンドではオブジェクトとメソッドについての基本的なコードコンプリートを提供します。Ctrl+スペースキー押下によりコンプリートメニューが出現します。（例：新規行で”V”を入力した時の場合、Vec2, Vec3, などの”V”から始まるオブジェクトがやってきます）このコードコンプリートはオブジェクトのメソッドやあるいはそれらのシグネチャもリストアップします。

ドラッグアンドドロップによる ポート順再配置

外部公開ポートを直接ドラッグアンドドロップしそれらを並び替えることがついに可能になりました。コンテキストメニューを呼び出す必要ありません。

Execute Ports

execute portsの使用法についてのサンプルショウケースを２つ同梱しています：

  • ForLoop.canvas では Execute.Merge4 ノードを Forノードで駆動し、異なる２つのサブグラフを実行します。

  • ForLoop_KochCurve.canvas ではコッホ雪片を描画、コッホ曲線の構築ルールにのっとり線分の反復再分割（と各ステップでの中間結果のvarへの保存）を行っています。Execute Ports についてのより詳しい情報は 明示的な Executeポート を参照してください。

Fabric for MODO

Fabric Engine 2.2 現在、MODO 統合版をついに公式サポートしインストールに含まれる形で配布します。Fabric for MODO プラグインは MODO 901 から MODO 10.0 まで互換性があり、 Windows, Linux and OS X で動作します。より詳しくは Fabric for MODO の文書および Fabric for MODO introductory video の紹介ビデオをご覧ください。

注釈

canvas.exe は canvas.py に 置き換わります

今後Canvasの起動には canvas.py ( Fabricスタンドアロン版の起動(canvas.py) に詳述)を使用するようにしてください。C++版の Canvas実行ファイルは非推奨となったためごく短い移行期間提供いたします。将来のバージョン、おそらく次のフィーチャーリリース版では削除します。Python版もC++も機能は同一です。(例外は新しい機能の Script Editor は canvas.py )

既知の不具合、制限事項、回避策¶

  • 制限: プリセットの保存にアタッチされたコメントが含まれない。 (FE-4900)

    回避策: 替わりにノードのツールチッププロパティを使用する

Fabric for MODO Limitations

  • Fabric の Inline Drawing をサポートするには MODOへの変更を必要とします。そのため私たちは Foundry とともに協同し、将来のMODO, Fabric for MODOのリリースで MODO のビューポートへ Fabric から描画可能な手段を提供を目指しています。

  • プロシージャルメッシュアイテム（CanvasPI）は、まだ入力メッシュとしてサポートしていません。（ただし出力は可能です）Fabric for MODO の将来のリリースにおいて、Input Mesh Port をサポートする予定です[FABMODO-29]

  • Per-polygon normal 未実装 [FABMODO-17]

  • 評価コンテキスト（evaluation context）のデータ、関連するプリセット（”EvalContext”）は Fabric for Modo には現時点で設定されていません。空の値を戻します。[FABMODO-15]
