Fabric Engine 2 リリースノート¶

Fabric Engine version 2.0.1
2015年11月 リリース
Copyright (c) 2010-2017 Fabric Software Inc. All rights reserved.

このリリースについて¶

Fabric Engine 2.0.1 はバグフィックスリリースです; Fabric Engine 2 リリースノート に 2.0.0 のより詳しい情報があります。(既知の制限など)

サポートするOS・DCC統合・GPU¶

  • Fabric 2 はWindows、Linux上で動作します。OS X 向けのビルドも提供されますが、公式にサポートされているわけではありません。今後のリリースによりサポート予定です。
  • Fabric for Maya 統合版は Autodesk Maya 2014、2015、2016（SP版含むWindows/Linux版）をサポートします。OS X は今後サポート予定です。サンプルシーンの中には Maya2016 並列実行モードを有効にしていると問題を生じる場合があります。これは Maya 2016 SP4で修正されています。
  • Fabric for Softimage 統合版は Autodesk Softimage 2014、2015（SP版含むWindows版）をサポートします。
  • Fabric 2 GPU supportは NVIDIAハードウェアから利用可能です。詳細はGPUコンピューティングガイドを御覧ください。

ドキュメント¶

Fabric Canvas について学びたいのであれば、こちらをどうぞ:

  • Canvas ユーザガイド
  • Canvas プログラマーガイド
  • Fabric Engine 入門

http://fabricengine.com/blog/ に記事と、how-to ビデオの紹介がありますのでチェックしてみてください。

ダウンロード¶

Fabric Engine 2 を取得するには http://dist.fabric-engine.com/FabricEngine/2.0.1/ から行えます。インストールについては Fabric Engine 入門 を参照してください。

サポート¶

Fabric 2 は評価版、スタジオ版、教育版が無償で利用可能です。また商用契約に基づく利用も可能です。商用ライセンスをご利用のお客様は、私達Fabricへ直接連絡いただけます。非商用のお客様はサポートについて以下のオプションをとることをお勧めいたします。

  • http://answers.fabricengine.com Fabric コミュニティによる Ask & answer questions。サポートに関する問題を投稿する場です。
  • https://groups.google.com/forum/#!forum/fabricengine より濃いディスカッションフォーラムです。Canvas グラフをシェアしましょう。
  • bugs@fabricengine.com もしバグを発見したら、明快な再現手段を付しEメールをください。(使用OSとDCCアプリ、それぞれのバージョンについての詳細も)Fabricからの返答は、Fabricプラットフォームの問題の解決にあたって追加情報を必要とする場合のみですので、ご了承ください。

変更点¶

注釈

KL2DFG の -inheritance, -polymorphism オプションでの動作は変更されました。以後 polymorphism フラグは polyThis と呼びます。新しい polyParams フラグを導入しました。これはエクステンション内のパラメータの型と同一の表記を持つメソッドに対し使用します。

注釈

There is also a change in the way node and preset names are handled. You can now change the node name of a preset, but it won’t be reflected in the node header, which will still show the preset name. Entering the preset will show the Path (which reflects the name of the node) and the Preset Name. When a node has been inlined (Split from preset), any references to the preset name are lost and it will only show the Path.

Fabric Engine 2.0.1 ではスタンドアロン向けの付属のサンプルを追加しました。

既知の不具合、制限事項、回避策¶

  • Limitation: Saving Presets won’t include the attached comment (FE-4900)

    回避策: 替わりにノードのツールチッププロパティを使用する

  • 制限: Fabric for Maya, Fabric for Softimage において、 「”Right-Click -> New Variable”」あるいは「既存の変数ノードをコピー・アンド・ペースト」し、新規の変数ノードを作成する際、ノードエクスプローラ中の変数リストが再更新されない (FE-5695)

    回避策: Canvas を一旦閉じ再度開くと変数がエクスプローラ上に現れます。

  • 制限: 「名前の変更のアンドゥ」がグラフエディタに反映されない(FE-5744)

    回避策: ノードに入る・閉じるで強制的にノード名に更新がかかります。

  • 制限: Maya用の surface scuplting サンプルシーン(manipulation_surfacesculpting.ma)が Maya 2016 で正常動作しない
